*autocmd* *autocmds* *hooks*

#AUTOCMDS#

Autocmds (automatic commands) execute shell commands in response to
specific events. They provide a hook system for customizing shell
behavior without modifying the shell itself.

==============================================================================
1. Registration                                *autocmd-register*

  `autocmd`
  `autocmd {kind}`
  `autocmd [OPTIONS] {kind} {command}`

  Register {command} to run whenever the event {kind} fires.

  With no arguments, prints every registered autocmd across all
  event kinds. With just a {kind} and no {command}, prints only the
  autocmds registered for that event. See the full list |here|(autocmd-list).

  Options:

    `-c`                                       *autocmd-clear*

        Clear all autocmds of the specified {kind}. No command is
        needed.

  Examples:

      `autocmd post-cmd 'echo "exit: $?"'`
      `autocmd post-change-dir 'echo "moved to $NEW_DIR"'`
      `autocmd -c pre-cmd`
      `autocmd`                  list every registered autocmd
      `autocmd post-cmd`         list autocmds registered for post-cmd

                                                 *autocmd-list*

  Listing ~

    `autocmd`           prints every registered autocmd, grouped
                        across all event kinds.

    `autocmd {kind}`    prints only the autocmds registered for that
                        event {kind}. Useful for inspecting what
                        will fire on a specific lifecycle event.

==============================================================================
2. Event Kinds                                 *autocmd-kinds*

  2.1 Command Execution                        *autocmd-cmd-events*

  `pre-cmd`                                    *autocmd-pre-cmd*

      Fires before a command is executed.

  `post-cmd`                                   *autocmd-post-cmd*

      Fires after a command finishes.

  `on-command-not-found`                       *autocmd-on-command-not-found*

      Fires if the command is not found.

      Special variables:
        `$CMD`        the command name that is not found

  2.2 Directory Changes                        *autocmd-dir-events*

  `pre-change-dir`                             *autocmd-pre-change-dir*

      Fires before |cd|(builtin-cd) changes the working directory.

      Special variables:
        `$NEW_DIR`    the directory being changed to
        `$OLD_DIR`    the current directory (before the change)

  `post-change-dir`                            *autocmd-post-change-dir*

      Fires after a successful directory change.

      Special variables:
        `$NEW_DIR`    the new working directory
        `$OLD_DIR`    the previous directory

  2.3 Job Events                               *autocmd-job-events*

  `on-job-finish`                              *autocmd-on-job-finish*

      Fires when a background job completes.

      Special variables:
        `$CHILD_COUNT`    number of commands in the job
        `$CHILDREN`       associative array mapping each command string in the
                          job to its exit status. Statuses are formatted as
                          plain integers for normal exits (0, 1, ...) and as
                          `128 + signum` for signaled exits (e.g. 130 for
                          SIGINT, 143 for SIGTERM).
        `$JOB_STATUS`     exit code of the last command in the job (same
                          encoding as values in `$CHILDREN`)
        `$JOB_ID`         the job table ID (positive integer)

  2.4 Prompt Events                            *autocmd-prompt-events*

  `pre-prompt`                                 *autocmd-pre-prompt*

      Fires before the |prompt| is rendered. Useful for updating prompt
      state.

  `post-prompt`                                *autocmd-post-prompt*

      Fires after the |prompt| is rendered.

  2.5 Mode Change Events                       *autocmd-mode-events*

  `pre-mode-change`                            *autocmd-pre-mode-change*

      Fires before the vi editing mode changes. The `$SHED_VI_MODE`
      variable still holds the old mode.

  `post-mode-change`                           *autocmd-post-mode-change*

      Fires after the vi editing mode changes. `$SHED_VI_MODE` reflects
      the new mode.

  2.6 History Events                           *autocmd-hist-events*

  `on-history-open`                            *autocmd-on-history-open*

      Fires when the fuzzy history search window opens.

      Special variables:
        `$ENTRIES`      array of all history entries
        `$NUM_ENTRIES`  count of all entries
        `$MATCHES`      array of currently matching entries
        `$NUM_MATCHES`  count of matching entries
        `$SEARCH_STR`   the current search string

  `on-history-close`                           *autocmd-on-history-close*

      Fires when the history search is dismissed without selecting.

  `on-history-select`                          *autocmd-on-history-select*

      Fires when a history entry is selected.

      Special variables:
        `$HIST_ENTRY`   the selected history entry

  2.7 Completion Events                        *autocmd-comp-events*

  `on-completion-start`                        *autocmd-on-completion-start*

      Fires when the completion menu becomes visible.

      Special variables:
        `$MATCHES`      array of completion candidates
        `$NUM_MATCHES`  count of candidates
        `$SEARCH_STR`   the token being completed

  `on-completion-cancel`                       *autocmd-on-completion-cancel*

      Fires when the completion menu is dismissed without selecting.

  `on-completion-select`                       *autocmd-on-completion-select*

      Fires when a completion candidate is accepted.

      Special variables:
        `$COMP_CANDIDATE`  the selected completion candidate

  2.8 Screensaver Events                       *autocmd-screensaver-events*

  `on-screensaver-exec`                        *autocmd-on-screensaver-exec*

      Fires when the screensaver activates after the idle timeout.

  `on-screensaver-return`                      *autocmd-on-screensaver-return*

      Fires when the shell returns from the screensaver command.

  2.9 Exit Event                               *autocmd-exit-event*

  `on-exit`                                    *autocmd-on-exit*

      Fires when the shell is about to exit.

  2.10 Time Report Event                       *autocmd-time-report*

  `on-time-report`                             *autocmd-on-time-report*

      Fires when a `time`-prefixed command completes. If no autocmd is
      registered for this event, the shell falls back to formatting
      with `$TIMEFMT`.

      Special variables:
        `$TIME_REAL_MS`    wall clock time in milliseconds
        `$TIME_USER_MS`    user CPU time in milliseconds
        `$TIME_SYS_MS`     system CPU time in milliseconds
        `$TIME_REAL_FMT`   wall clock time, formatted
        `$TIME_USER_FMT`   user CPU time, formatted
        `$TIME_SYS_FMT`    system CPU time, formatted
        `$TIME_CPU_PCT`    CPU percentage ((user+sys)/real*100)
        `$TIME_RSS`        peak resident set size in KB
        `$TIME_CMD`        the command that was timed

==============================================================================
3. Context Variables                           *autocmd-variables*

  Many autocmd events expose special variables within the scope of the
  autocmd execution. These variables are only available inside the
  autocmd command and are not set globally.

  Example:

      `autocmd post-change-dir '`
      `    if [[ "$NEW_DIR" == */projects/* ]]; then`
      `        echo "entering project directory"  `
      `    fi`
      `'`

==============================================================================
4. Behavior                                    *autocmd-behavior*

  - Multiple autocmds can be registered for the same event kind. They
    execute in registration order.

  - If an autocmd command fails, the error is printed but subsequent
    autocmds for the same event still run.

  - Autocmds do not affect the shell's exit status (`$?`). The exit
    status is saved before autocmd execution and restored afterward.

  - Special variables are only available within the scope of the
    autocmd execution. They are not set globally.

==============================================================================
5. Examples                                    *autocmd-examples*

  Notify on directory change:

      `autocmd post-change-dir 'echo "moved to $NEW_DIR"'`

  Custom time report with color (falls back to plain when piped):

      `_time_func() {`
      `    if [ -t 0 ]; then`
      `        echo`
      `        echo -e "\e[36;1m$TIME_CMD"`
      `        echo -e "\e[36;1mreal\e[0m\t$TIME_REAL_FMT"`
      `        echo -e "\e[36;1muser\e[0m\t$TIME_USER_FMT"`
      `        echo -e "\e[36;1msys\e[0m\t$TIME_SYS_FMT"`
      `        echo`
      `    else`
      `        echo`
      `        echo "$TIME_CMD"`
      `        echo -e "real\t$TIME_REAL_FMT"`
      `        echo -e "user\t$TIME_USER_FMT"`
      `        echo -e "sys\t$TIME_SYS_FMT"`
      `        echo`
      `    fi`
      `}`
      `autocmd on-time-report _time_func`

  Refresh prompt on mode change (for mode indicator):

      `autocmd post-mode-change 'kill -USR1 $$'`
      (SIGUSR1 can be used to remotely refresh the prompt. See |prompt|)

  Log completed jobs:

      `autocmd on-job-finish 'echo "job done" >> /tmp/jobs.log'`

  Clean up on exit:

      `autocmd on-exit 'rm -f /tmp/my-shell-*.tmp'`
