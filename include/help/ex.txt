*ex* *ex-mode* *ex-commands* *colon-commands*

#EX MODE#

Ex mode provides colon commands for operations that go beyond single-key
normal mode actions. Enter ex mode by pressing `:` in normal mode.

The command line supports full editing via insert mode, and has its own
command history navigable with `Up` and `Down`.

==============================================================================
1. Line Addresses                              *ex-address* *ex-range*

  Most ex commands accept an optional address or range before the command
  name to specify which lines to operate on.

  Single address forms:

    `{n}`         line number (1-indexed). `1` is the first line.
    `.`           the current line
    `$`           the last line
    `.+{n}`       current line plus offset (e.g. `.+2`)
    `.-{n}`       current line minus offset
    `/{pat}/`     next line matching pattern {pat} (wraps around)
    `?{pat}?`     previous line matching pattern {pat} (wraps around)

  Range forms (two addresses separated by `,`):

    `{a},{b}`     lines {a} through {b} inclusive
    `%`           shorthand for `1,$` (all lines)

  Examples:

    `:3d`         delete line 3
    `:1,5d`       delete lines 1 through 5
    `:.,$y`       yank from current line to end of buffer
    `:%s/foo/bar` substitute on all lines

  If no address is given, the command operates on the current line (or
  all lines, for commands like `:s` with `%`).

==============================================================================
2. Substitute                                  *ex-substitute* *ex-s*

  `:s/{old}/{new}/[flags]`                     *ex-sub*

      Replace occurrences of {old} (a regular expression) with {new}
      on the addressed lines. Without a range, operates on the current
      line.

      Flags:

        `g`   replace all occurrences on each line (default: first only)
        `i`   case-insensitive match
        `I`   case-sensitive match (overrides `i`)
        `n`   show count of matches without substituting

      The delimiter is typically `/` but any non-alphanumeric character
      may be used (e.g. `:s|foo|bar|`). Delimiters can be escaped with
      a backslash inside the pattern.

      Examples:

        `:s/foo/bar/`       replace first `foo` with `bar` on current line
        `:s/foo/bar/g`      replace all `foo` with `bar` on current line
        `:%s/foo/bar/g`     replace all `foo` with `bar` in entire buffer
        `:1,5s/^/#/`        prepend `#` to lines 1-5

  `:s`                                         *ex-repeat-sub*

      Repeat the last substitute command. If a range is given (e.g.
      `:%s`), the repeat is applied to those lines instead of the
      range from the original command.

==============================================================================
3. Global                                      *ex-global* *ex-g*

  `:g/{pat}/{cmd}`                             *ex-global-cmd*

      Execute ex command {cmd} on every line matching pattern {pat}.
      The default range is the entire buffer.

      Examples:

        `:g/foo/d`              delete all lines containing `foo`
        `:g/^#/s/^#//`         remove leading `#` from all comment lines
        `:g/error/normal!dd`   delete lines containing "error" via normal!

  `:g!/{pat}/{cmd}`                            *ex-global-not*

      Execute {cmd} on every line that does NOT match {pat}. Equivalent
      to `:v/{pat}/{cmd}` in traditional vi.

  `:g`                                         *ex-repeat-global*

      Repeat the last global command.

==============================================================================
4. Normal Mode Commands                        *ex-normal*

  `:normal!{seq}`                              *ex-normal-cmd*

      Execute {seq} as a sequence of normal mode keystrokes. The `!`
      is required. When used with a line range, the sequence is applied
      to each line in the range independently, with the cursor starting
      at column 0 of each line.

      All operations within a ranged `:normal!` are grouped into a
      single undo operation.

      Examples:

        `:normal!dw`            delete first word on current line
        `:1,5normal!>>` indent lines 1-5
        `:g/foo/normal!A;`      append `;` to end of every line containing `foo`

==============================================================================
5. Shell Commands                              *ex-shell*

  `:!{cmd}`                                    *ex-bang*

      Execute {cmd} in the shell. The following special variables are
      set during execution and can be read or modified:

        `$BUFFER`    the current editor buffer contents
        `$CURSOR`    the cursor position (flat byte index)
        `$ANCHOR`    the visual selection anchor position

      If the command modifies these variables, the editor state is
      updated accordingly. This allows ex commands to programmatically
      edit the buffer.

      If the command sets `$KEYS`, the value is fed back into the
      editor as a key sequence.

      Example:
        `:!echo "$BUFFER" | tr a-z A-Z > /tmp/out`
        `:!BUFFER=$(echo "$BUFFER" | sort)`

  `:{range}!{cmd}`                              *ex-bang-filter*

      Filter lines through {cmd}. The addressed lines are piped to
      {cmd} as stdin, and replaced with its stdout. This is the same
      as `vim`'s filter command.

      With a single line address:

        `:5!tr a-z A-Z`          uppercase line 5
        `:.!date`                replace current line with date output

      With a range:

        `:1,5!sort`              sort lines 1 through 5
        `:%!sort -u`             sort and deduplicate the entire buffer
        `:'<,'>!column -t`       align the visual selection into columns

      If the command produces no output, the addressed lines are
      deleted.

      This is particularly powerful with shell functions:

        `:%!indent_code`         pipe entire buffer through a function
        `:3,10!awk '{print NR, $0}'`  number lines 3-10

==============================================================================
6. File Operations                             *ex-file*

  Reading a file with `:r {file}` marks it as the editor's "active
  file." Subsequent `:w`, `:wq`, and `:x` commands target the active
  file by default if no path is given. The shell-exit verbs (`:q`,
  `:wq`) also change behavior when an active file is set: instead of
  exiting the shell they clear the editor back to a fresh prompt.

  The active file's path is exposed to the shell as `$EDITOR_FILE`
  (empty string when no file is active).

  `:r {file}`                                  *ex-read*

      Read the contents of {file} and insert them into the buffer at
      the cursor position. Marks {file} as the active file so that
      subsequent `:w`/`:wq`/`:x` can be issued without arguments.

  `:r !{cmd}`                                  *ex-read-cmd*

      Execute {cmd} and insert its output into the buffer. Does NOT
      set an active file.

  `:w`                                         *ex-write-active*

      Write the buffer contents to the active file (the one last
      opened with `:r {file}`). Errors if no active file is set.

  `:w {file}`                                  *ex-write*

      Write the buffer contents to {file}. Creates the file if it does
      not exist, or truncates it if it does. Does not change the
      active file.

  `:w >> {file}`                               *ex-write-append*

      Append the buffer contents to {file}.

  `:w !{cmd}`                                  *ex-write-cmd*

      Pipe the buffer contents to {cmd} as stdin.

  `:wq`                                        *ex-write-quit*
  `:x`                                         *ex-x*

      Write the buffer (to the active file if no path is given) and
      then clear the editor. If no active file is set, behaves as
      `:q` and exits the shell.

      `:x` is a synonym for `:wq`.

  `:e {file}`                                  *ex-edit*

      Open {file} in the editor defined by `$EDITOR`. Requires the
      `EDITOR` environment variable to be set. Unlike `:r`, this
      spawns an external editor rather than loading into shed's own
      buffer, so it does NOT set an active file.

      Example:
        `:e ~/.config/shed/shedrc`

==============================================================================
7. Buffer Operations                           *ex-buffer*

  `:d`                                         *ex-delete*

      Delete the addressed lines. Without a range, deletes the current
      line. `:%d` clears the entire buffer.

  `:y`                                         *ex-yank*

      Yank the addressed lines into the default register. Without a
      range, yanks the current line.

  `:pu`                                        *ex-put*

      Put (paste) from the default register after the cursor.

==============================================================================
8. Expand                                      *ex-expand*

  `:expand`                                    *ex-expand-cmd*

      Perform shell expansion on the addressed lines, replacing them
      in-place with the expanded result. Variables (`$VAR`,
      `${param}`), command substitutions (`$(...)`, ``...``), tildes
      (`~`, `~user`), arithmetic (`$((...))`), and brace expansion
      (`{a,b,c}`) are all expanded.

      Quoting structure is preserved: a value that was inside double
      quotes stays inside double quotes after expansion, and operators
      (`&&`, `|`, `;`, etc.) are left as operators. The result is a
      valid shell command line that, when re-executed, behaves the
      same as the original.

      Default range is the entire buffer (equivalent to `:%expand`).
      To expand only the current line, use `:.expand`.

      Examples:

        `:expand`           expand the entire buffer
        `:.expand`          expand just the current line
        `:3,5expand`        expand lines 3 through 5

  `:expand!`                                   *ex-expand-bang*

      Perform expansion with full POSIX shell semantics: word
      splitting and globbing are applied, quotes are removed, and the
      output is the literal text the shell would hand to the kernel
      after all expansion phases.

      Use this when you need the fully-resolved form (e.g. to paste
      into a Makefile, log message, or documentation) rather than an
      editor-friendly equivalent.

      Examples:

        `:expand!`          full expansion of the entire buffer
        `:.expand!`         full expansion of the current line

      The bangless form (`:expand`) is round-trip safe: re-executing
      the result produces the same effect as re-executing the
      original. The bang form (`:expand!`) collapses shell syntax
      into literal data, which may not be re-executable if any
      expanded value contains shell metacharacters.

==============================================================================
9. Other Commands                              *ex-other*

  `:q`                                         *ex-quit*

      If no active file is set (see |ex-file|), exits the shell.

      If an active file is set, clears the editor buffer and returns
      to a fresh prompt without writing. The shell stays running.

      Use this distinction when you've opened a file with `:r {file}`:
      `:q` lets you discard your edits and get back to the prompt
      without leaving shed.

  `:help {topic}`                              *ex-help*

      Display help for {topic}. Runs the `help` builtin.

==============================================================================
10. Path Expansion                             *ex-paths*

  File paths in ex commands are subject to variable expansion. You can
  use environment variables in paths:

      `:e $HOME/.config/shed/shedrc`
      `:w ${TMPDIR}/output.txt`

==============================================================================
11. Ex Command History                         *ex-history*

  Ex mode maintains its own command history, separate from the main
  shell history. Navigate with `Up` and `Down` while in ex mode, or
  open history search with `Ctrl+R`.

==============================================================================
12. Stash                                      *ex-stash*

  The stash lets you save and recall command snippets directly
  from the line editor. Unnamed stashes act as a temporary stack;
  named stashes are persistent snippets.

  `:stash`                                      *ex-stash-push*

      Push the current buffer and cursor position onto the unnamed
      stack, then clear the buffer.

  `:stash {name}`

      Save the current buffer and cursor position as a named stash.
      If {name} already exists, it is overwritten.

  `:stash pop [N]`                              *ex-stash-pop*

      Pop the most recent unnamed stash (or index {N}) and replace
      the buffer with it. The cursor is restored to the saved
      position.

  `:stash apply {name}`                         *ex-stash-apply*

      Apply a named stash to the buffer. The stash remains saved
      for future use.

  `:stash insert {name}`                        *ex-stash-insert*

      Insert a named stash at the current cursor position. The
      saved cursor offset is applied relative to the insertion
      point.

  `:stash swap [N | {name}]`                    *ex-stash-swap*

      Exchange the current buffer with a stashed entry. The
      buffer takes on the stashed content; the stash slot takes
      on what the buffer used to hold.

      For named entries, swap is its own inverse: running
      `:stash swap wip` twice returns you to where you started.

      For indexed (unnamed) entries, the swapped-out buffer is
      pushed to the top of the stack and the targeted row is
      removed. As a side effect, repeated `:stash swap {N}`
      rotates the buffer through every entry from index `N` up
      to the top, returning to the original state after
      `(stack_len - N + 1)` invocations.

      With no argument, defaults to the top of the stack.

  `:stash drop [N | {name}]`                    *ex-stash-drop*

      Remove a stash entry by index or name.

  `:stash list [stack | named]`                 *ex-stash-list*

      List stashed entries. Optionally filter to only stack or
      only named entries.

  Named stashes can be predefined in your `shedrc` using the
  |stash|(builtin-stash) builtin:

      `stash --save commit 'git commit -m ""' 16`

  Then recalled instantly from the editor:

      `:stash apply commit`
