*builtin* *builtins* *builtin-commands*

#BUILTIN COMMANDS#

Builtins are commands implemented directly by the shell rather than as
external executables. They have access to shell state (variables, job table, etc.) that external commands cannot reach.

The following builtins are available:

  `alias`       define or list command aliases              |builtin-alias|
  `arrops`      array manipulation (push, pop, rotate)      |builtin-arrops|
  `autocmd`     register hooks for shell events             |builtin-autocmd|
  `bg`          resume a stopped job in the background      |builtin-bg|
  `cd`          change the working directory                |builtin-cd|
  `complete`    define custom tab completion rules          |builtin-complete|
  `declare`     declare variables with type and attributes  |builtin-declare|
  `defer`       register cleanup tied to scope exit         |builtin-defer|
  `dirs`        display the directory stack                 |builtin-dirs|
  `disown`      remove a job from the job table             |builtin-disown|
  `echo`        print arguments to stdout                   |builtin-echo|
  `eval`        evaluate a string as shell input            |builtin-eval|
  `exec`        replace the shell process with a command    |builtin-exec|
  `exit`        exit the shell                              |builtin-exit|
  `export`      mark variables for export to environment    |builtin-export|
  `false`       return exit status 1                        |builtin-false|
  `fc`          edit and re-exec commands from history      |builtin-fc|
  `fg`          bring a job to the foreground               |builtin-fg|
  `getopts`     parse option flags in scripts               |builtin-getopts|
  `hash`        manage the command location cache           |builtin-hash|
  `help`        display help pages                          |builtin-help|
  `hist`        query and manage shell history              |builtin-hist|
  `jobs`        list background jobs                        |builtin-jobs|
  `keymap`      define custom key bindings                  |builtin-keymap|
  `kill`        send signals to processes                   |builtin-kill|
  `local`       declare variables local to a function       |builtin-local|
  `map`         structured key-value store                  |builtin-map|
  `msg`         post a message to the status bar            |builtin-msg|
  `popd`        pop the directory stack                     |builtin-popd|
  `pushd`       push a directory onto the stack             |builtin-pushd|
  `pwd`         print the working directory                 |builtin-pwd|
  `read`        read a line of input into a variable        |builtin-read|
  `readkey`     read a single keypress                      |builtin-readkey|
  `readonly`    mark variables as read-only                 |builtin-readonly|
  `return`      return from a function                      |builtin-return|
  `break`       exit a loop                                 |builtin-break|
  `continue`    skip to the next loop iteration             |builtin-continue|
  `seek`        seek a file descriptor to a byte offset     |builtin-seek|
  `set`         set shell options and positional parameters |builtin-set|
  `stash`       save and recall command snippets            |builtin-stash|
  `shift`       shift positional parameters                 |builtin-shift|
  `shopt`       get or set shell options                    |builtin-shopt|
  `source`      execute a file in the current shell         |builtin-source|
  `[[`          evaluate conditional expressions            |builtin-test|
  `times`       print shell and child CPU times             |builtin-times|
  `trap`        set signal handlers                         |builtin-trap|
  `true`        return exit status 0                        |builtin-true|
  `type`        describe how a name would be interpreted    |builtin-type|
  `ulimit`      set process resource limits                 |builtin-ulimit|
  `umask`       set the file creation mask                  |builtin-umask|
  `unalias`     remove aliases                              |builtin-unalias|
  `unset`       remove variables or functions               |builtin-unset|
  `wait`        wait for background jobs to finish          |builtin-wait|

==============================================================================
ALIAS                                            *builtin-alias* *alias*

  `alias [name[=value] ...]`

  With no arguments, lists all defined aliases in alphabetical order,
  in the form `name="value"`. Alias definitions printed this way are
  valid shell input that can be reused to recreate the alias.

  With arguments of the form `name=value`, defines an alias. When
  {name} is used as a command, it is replaced with {value} before
  execution. Multiple aliases can be defined in a single call.

  Alias expansion occurs very early in the execution process.

  With an argument that has no `=`, prints the current definition of
  that alias.

  The names `command` and `builtin` are reserved and cannot be aliased.

  Examples:

      `alias`                           list all aliases
      `alias ll='ls -la'`               define an alias
      `alias ll`                        print: ll='ls -la'
      `alias g='git' gs='git status'`   define multiple

==============================================================================
UNALIAS                                          *builtin-unalias* *unalias*

  `unalias [name ...]`

  Remove the named aliases. With no arguments, lists all defined
  aliases (same as `alias` with no arguments).

  It is an error to unalias a name that is not currently defined.

  Examples:

      `unalias ll`               remove the ll alias
      `unalias g gs`             remove multiple aliases

==============================================================================
ARROPS                                           *builtin-arrops* *arrops*

  Array operation builtins for manipulating shell |arrays|. All commands
  share a common set of options:

    `-c {n}`     perform the operation {n} times (default: 1)
    `-v {var}`   store the result in {var} instead of printing to stdout
                 (pop/fpop only)
    `-r`         reverse direction (rotate only)

  PUSH                                           *builtin-push* *push*

  `push {array} {value} ...`

      Append one or more values to the back of {array}. If {array}
      does not exist, it is created. Multiple values are appended
      left to right.

      Examples:
          `push myarr a b c`     append a, b, c to the back

  FPUSH                                          *builtin-fpush* *fpush*

  `fpush {array} {value} ...`

      Prepend one or more values to the front of {array}. Each value
      is pushed to the front sequentially, so the last value ends up
      at index 0.

      Example:
          `fpush arr a b`    given arr=[c], result is [b, a, c]

  POP                                            *builtin-pop* *pop*

  `pop [-c {n}] [-v {var}] {array}`

      Remove and print the last element of {array}. With `-c`, removes
      that many elements from the back, printing each on its own line.
      With `-v`, stores the popped value in {var} instead of printing.

      Sets exit status 1 if the array is empty or does not exist.

      Examples:
          `pop myarr`            print and remove last element
          `pop -c 2 myarr`       remove and print last 2 elements
          `pop -v x myarr`       store last element in $x

  FPOP                                           *builtin-fpop* *fpop*

  `fpop [-c {n}] [-v {var}] {array}`

      Remove and print the first element of {array}. Equivalent to
      `pop` but operates on the front of the array.

      Examples:
          `fpop myarr`           print and remove first element
          `fpop -c 2 myarr`      remove and print first 2 elements
          `fpop -v x myarr`      store first element in $x

  ROTATE                                         *builtin-rotate* *rotate*

  `rotate [-c {n}] [-r] {array}`

      Rotate {array} left by {n} positions (default: 1). With `-r`,
      rotate right instead. If {n} exceeds the array length it is
      clamped, making a full rotation a no-op.

      Examples:
          `rotate arr`           [a,b,c,d] -> [b,c,d,a]
          `rotate -c 2 arr`      [a,b,c,d] -> [c,d,a,b]
          `rotate -r arr`        [a,b,c,d] -> [d,a,b,c]

AUTOCMD                                          *builtin-autocmd* *autocmd*

  `autocmd [-c] {event} [{command}]`

  Register {command} to run automatically when {event} fires. Multiple
  commands can be registered for the same event; they run in
  registration order. The exit status of autocmd handlers does not
  affect `$?`.

  `-c`    clear all registered commands for {event} instead of adding one

  Available events:

    `pre-cmd`               before a command is executed
    `post-cmd`              after a command completes
    `pre-change-dir`        before `cd` changes the directory
    `post-change-dir`       after `cd` changes the directory
    `on-job-finish`         when a background job finishes
    `pre-prompt`            before the prompt is displayed
    `post-prompt`           after the prompt is displayed
    `pre-mode-change`       before the editor mode changes
    `post-mode-change`      after the editor mode changes
    `on-history-open`       when history search opens
    `on-history-close`      when history search closes
    `on-history-select`     when a history entry is selected
    `on-completion-start`   when tab completion starts
    `on-completion-cancel`  when tab completion is cancelled
    `on-completion-select`  when a completion candidate is selected
    `on-screensaver-exec`   when the screensaver command starts
    `on-screensaver-return` when the screensaver command exits
    `on-time-report`        when a timed command reports its duration
    `on-exit`               when the shell exits

  Examples:

      `autocmd post-cmd 'echo done'`
      `autocmd post-change-dir 'ls'`
      `autocmd -c post-cmd`           clear all post-cmd handlers

  For special variables available inside autocmd handlers, see |autocmd|.

CD                                               *builtin-cd* *cd*

  `cd [-L|-P] [{dir}]`

  Change the working directory to {dir}. With no argument, changes to
  `$HOME`. Updates `$PWD` and `$OLDPWD`.

  `-L`    follow symbolic links (default)
  `-P`    resolve symbolic links to their physical (canonical) path

  Special arguments:

    `-`    change to `$OLDPWD` (the previous directory) and print the
           new path. Repeating `cd -` toggles between the two directories.

  `$CDPATH`

      A colon-separated list of directories to search when {dir} is a
      relative path that does not begin with `./`, or `../`. An
      empty entry in `$CDPATH` is treated as `.`.

  Fires the `pre-change-dir` and `post-change-dir` autocmd events.
  See |autocmd| for available hook variables.

  Examples:

      `cd`                  go to $HOME
      `cd /tmp`             absolute path
      `cd -`                return to previous directory
      `cd -P ~/link`        resolve symlink before changing

COMPLETE                                         *builtin-complete* *complete*

  `complete [options] {cmd} ...`

  Register a tab completion specification for one or more commands. When
  the user presses Tab after {cmd}, the shell uses the registered spec to
  generate candidates. Multiple commands may be given; they all receive
  the same spec.

  Source Flags (what to complete) ~

    `-W {wordlist}`   complete from a static whitespace-separated word list
    `-F {func}`       call shell function {func} to generate candidates.
                      The function should populate `$COMPREPLY` with the
                      list of matches.
    `-A {action}`     complete from a built-in action name (see below)
    `-f`              complete file names
    `-d`              complete directory names
    `-c`              complete command names (executables and builtins)
    `-u`              complete user names
    `-v`              complete variable names
    `-a`              complete alias names
    `-j`              complete job names
    `-S`              complete signal names

  Behavior Flags ~

    `-o default`      fall back to default (filename) completion if the
                      spec produces no candidates
    `-o dirnames`     fall back to directory completion if no candidates
    `-o space`        append a space after a completed word (default)

  Management Flags ~

    `-p [{cmd} ...]`  print registered specs. With no {cmd}, prints all.
    `-r [{cmd} ...]`  remove the spec for each {cmd}.

  Examples:

      `complete -W 'start stop restart' myservice`
      `complete -F _git_complete git`
      `complete -f -d cp`
      `complete -p git`           print git's registered spec
      `complete -r git`           remove git's spec

  When `-F` is used, the following variables are set before the function
  is called and should be read (not modified) by the function:

    `$COMP_WORDS`     array of words on the current command line
    `$COMP_CWORD`     index of the word being completed
    `$COMP_LINE`      the full current command line as a string
    `$COMP_POINT`     cursor position within `$COMP_LINE`

  The function sets `$COMPREPLY` to an array of completion candidates.

==============================================================================
COMPGEN                                          *builtin-compgen* *compgen*

  `compgen [options] [{prefix}]`

  Generate completion candidates without registering a spec. Applies the
  given source flags against {prefix} (if provided) and prints one
  candidate per line. Useful for testing completion logic or inside `-F`
  functions.

  Accepts the same source flags as `complete` (`-W`, `-F`, `-f`, `-d`,
  `-c`, `-u`, `-v`, `-a`, `-j`, `-S`, `-o`).

  Examples:

      `compgen -W 'foo bar baz'`        print all three words
      `compgen -W 'foo bar baz' b`      print only: bar baz
      `compgen -f /tmp/`                list files under /tmp/
      `compgen -v HOME`                 list variables starting with HOME
      `compgen -F _my_func`             run function and print results
      `compgen -o default`              fall back to default completion

DECLARE                                          *builtin-declare* *declare*

  `declare [-aAirx] [name[=value] ...]`
  `declare -p [name ...]`
  `declare -f [name ...]`
  `declare -F [name ...]`

  Declare variables, optionally setting their type and attributes in
  the same step. With no flags and a `name=value` argument, behaves
  like a plain assignment. Combined with `-a`, `-A`, `-i`, `-r`, or
  `-x`, fixes the variable's type and/or attributes at declaration
  time.

  With no arguments and no introspection flag, lists all variables in
  the current scope as `declare`-syntax assignments suitable for
  re-execution.

  Type flags ~

    `-a`    indexed array. RHS may be `(elem1 elem2 ...)`
    `-A`    associative array. RHS may be `([key1]=v1 [key2]=v2 ...)`
    `-i`    integer. RHS is arithmetic-evaluated, so `declare -i x=5+3`
            stores `8`. Subsequent assignments are also evaluated.

  Only one type flag may be given per call. Without a type flag, the
  variable is a string.

  Attribute flags ~

    `-r`    read-only. Equivalent to `readonly`. Cannot be reassigned
            or unset. See |builtin-readonly|.
    `-x`    export. Equivalent to `export`. Marks the variable for
            propagation to child processes. See |builtin-export|.

  Type and attribute flags can be combined: `declare -ix N=10` makes
  `N` an integer that is also exported.

  Introspection flags ~

    `-p`    print all variables (or only the named ones) as
            `declare`-syntax assignments. Suitable for re-execution.
    `-f`    print all functions (or only the named ones) including
            their bodies, as parseable function definitions.
    `-F`    print only the function names (or only the named ones),
            one per line in the form `declare -f {name}`. Useful for
            checking whether a function is defined without dumping its
            body.

  Introspection flags are mutually exclusive with each other and with
  type/attribute flags.

  Scope ~

  `declare` operates on the current scope, the same way |builtin-local|
  does. Inside a function or brace group, the variable is local to
  that scope and is discarded when the scope exits. At the top level,
  it is global.

  Examples:

      `declare`                       list all variables in scope
      `declare x=10`                  string variable
      `declare -i n=5+3`              integer, evaluates to 8
      `declare -a arr=(a b c)`        indexed array
      `declare -A m=([k1]=v1 [k2]=v2)` associative array
      `declare -rx PATH`              mark existing var read-only and exported
      `declare -p PATH`               print PATH as a declare statement
      `declare -F`                    list all defined function names
      `declare -f my_func`            print the body of my_func

  See also: |builtin-local|, |builtin-export|, |builtin-readonly|,
  |builtin-unset|.

DEFER                                            *builtin-defer* *defer*

  `defer [-c] {command} ...`

  Register {command} to run when the current scope exits. Multiple
  arguments are joined into a single command string. Deferred commands
  run in LIFO order (most recently registered first) regardless of
  whether the scope exits normally, via `return`, `break`/`continue`,
  or by falling out of a brace group.

  Scopes that fire defers ~

    Function bodies                    fires on function return
    Brace groups `{ ... }`             fires on block exit
    Compond commands (`if`, `for`,...) fires on exiting the body
    The shell itself (top level)       fires on shell exit

  Options ~

    `-c`         clear all currently-registered defers in the current
                 scope before registering the new one

  With no arguments, lists the deferred commands registered in the
  current scope, one per line.

  Defer pairs naturally with |builtin-local| for scope-bound RAII
  patterns: declare a `local` to hold a saved value, then `defer` the
  restoration command. Both die together when the scope ends.

  Examples:

      `defer 'echo bye'`                              run on scope exit
      `defer "rm -rf '$tmpdir'"`                      cleanup
      `defer "shopt core.nullglob=$(shopt core.nullglob)"`  snapshot+restore
      `defer -c 'replacement'`                        drop all, register one
      `defer`                                         list current defers

DIRS                                             *builtin-dirs* *dirs*

  `dirs [-clpv] [+N | -N]`

  Display the directory stack. The current working directory is always
  at the top (index 0); entries pushed by `pushd` follow.

  `-c`       clear the directory stack
  `-l`       show full paths. Do not abbreviate `$HOME` as `~`
  `-p`       print one entry per line
  `-v`       print one entry per line, preceded by its stack index
  `+N`       print only the entry at position N from the top (0-based)
  `-N`       print only the entry at position N from the bottom

  With no arguments, prints all entries on one line separated by spaces,
  abbreviating `$HOME` as `~`.

  Examples:

      `dirs`              list the full directory stack
      `dirs -v`           list with numeric indices
      `dirs +0`           print only the current directory
      `dirs -c`           clear the stack

==============================================================================
ECHO                                             *builtin-echo* *echo*

  `echo [-n] [-e] [-E] [-p] [arg ...]`

  Write {arg}s to standard output, separated by spaces, followed by a
  newline.

  `-n`    suppress the trailing newline
  `-e`    interpret backslash escape sequences (see below)
  `-E`    disable escape interpretation (overrides `xpg_echo`)
  `-p`    expand each argument as a prompt string (see |prompt|)

  Escape sequences (active with `-e` or when `shopt core.xpg_echo=true`):

    `\n`     newline
    `\t`     horizontal tab
    `\r`     carriage return
    `\a`     alert (bell)
    `\b`     backspace
    `\e`     escape character (ESC)
    `\\`     literal backslash
    `\xNN`   byte with hex value NN (up to 2 digits)
    `\0NNN`  byte with octal value NNN (up to 3 digits)

  By default escape sequences are NOT interpreted. Set
  `shopt core.xpg_echo=true` to change this globally; `-E` opts out.

  Examples:

      `echo hello world`
      `echo -n no newline`
      `echo -e 'line1\nline2'`

==============================================================================
EVAL                                             *builtin-eval* *eval*

  `eval [arg ...]`

  Join all {arg}s with spaces and execute the result as shell input in
  the current environment. Variable assignments, function definitions,
  and other side effects persist after `eval` returns.

  With no arguments, succeeds silently.

  Examples:

      `eval echo hello`
      `eval 'x=42; echo $x'`
      `cmd='echo evaluated'; eval $cmd`

==============================================================================
EXEC                                             *builtin-exec* *exec*

  `exec [command [arg ...]]`

  Replace the current shell process with {command}. On success `exec`
  does not return. The shell is replaced entirely.

  With no arguments, succeeds silently (useful for applying redirections
  to the current shell without replacing it).

  It is an error if {command} cannot be found or executed.

  Examples:

      `exec /bin/sh`
      `exec myprogram arg1`

==============================================================================
EXIT                                             *builtin-exit* *exit*

  `exit [n]`

  Exit the shell with status {n}. If {n} is omitted, exits with status
  0. {n} must be a decimal integer.

  Examples:

      `exit`        exit with current status
      `exit 0`      exit successfully
      `exit 1`      exit with failure

EXPORT                                           *builtin-export* *export*

  `export [name[=value] ...]`

  Mark variables for automatic export to the environment of subsequently
  executed commands. With `name=value`, sets and exports in one step.
  With just {name}, marks an existing variable as exported.

  With no arguments, lists all current environment variables
  alphabetically as `name="value"` pairs.

  Examples:

      `export`                  list all environment variables
      `export PATH`             export an existing variable
      `export EDITOR=nvim`      set and export
      `export A=1 B=2`          multiple in one call

FC                                               *builtin-fc* *fc*

  `fc [-e editor] [-nlrs] [first [last]]`
  `fc -s [old=new] [first]`

  Operate on history entries. {first} and {last} select a range and can
  be a history number, a negative offset from the most recent entry
  (e.g. `-2`), or a string prefix matching a command.

  Options ~

    `-e {editor}`   open selected entries in {editor} for editing, then
                    re-execute. Uses `$FCEDIT` or `$EDITOR` if not given.
    `-l`            list entries instead of editing. Defaults to the
                    most recent 16 entries when no range is given.
    `-n`            suppress line numbers in `-l` output
    `-r`            reverse the order of selected entries
    `-s [old=new]`  re-execute without opening an editor. If `old=new`
                    is given, substitute {old} with {new} first.

  After an edit session, if the command changed it is appended to
  history; if unchanged it is not. Per POSIX, `fc` itself is not
  recorded in history when it edits and re-executes a command.

  Examples:

      `fc -l`                   list last 16 commands
      `fc -l 1 10`              list entries 1-10
      `fc -s echo`              re-exec last command starting with "echo"
      `fc -s cat=dog`           re-exec with cat -> dog substitution
      `fc -e vim`               edit last command in vim

==============================================================================
GETOPTS                                          *builtin-getopts* *getopts*

  `getopts {optstring} {name} [arg ...]`

  Parse short option flags. Intended for use inside a `while` loop.
  Processes positional parameters by default, or {arg}s if provided.

  {optstring} is a string of recognized option letters. A letter
  followed by `:` requires an argument. A leading `:` enables silent
  error handling.

  Each call places the next recognized option in {name}. If the option
  takes an argument, it is placed in `$OPTARG`. `$OPTIND` tracks the
  position of the next argument. Returns exit status 1 when no more
  options remain.

  Examples:

      `while getopts 'ab:c' opt; do`
      `  case $opt in`
      `    a) echo "got -a" ;;`
      `    b) echo "got -b $OPTARG" ;;`
      `    c) echo "got -c" ;;`
      `  esac`
      `done`

HASH                                             *builtin-hash* *hash*

  `hash [-r] [--refresh] [name ...]`

  Manage the command path cache. When `hashall` is enabled, the shell
  caches the full path of commands it has looked up.

  With no arguments, prints all cached commands as `name=path` pairs.
  With {name} arguments, explicitly adds those commands to the cache
  (error if not found in `$PATH`).

  `-r`          clear the entire cache
  `--refresh`   rescan `$PATH` and rebuild the cache

  Examples:

      `hash`             list cached paths
      `hash git nvim`    cache specific commands
      `hash -r`          clear the cache
      `hash --refresh`   rebuild from $PATH

HELP                                             *builtin-help* *help*

  `help [{topic}]`

  Display help for {topic}. With no argument, opens the main help index.
  {topic} may be a named help page, or anything that matches the tags
  on individual topics inside the help pages.

  If the `--list-tags` or `-l` flags are passed, lists all searchable
  help topics.

  Examples:

      `help`             open the help index
      `help --list-tags` list all searchable tags
      `help echo`        help for the echo builtin
      `help param`       parameter expansion reference

HIST                                             *builtin-hist* *hist*

  `hist [options] [id ...]`

  Query and manage shell history. History is stored in a SQLite
  database and supports rich filtering. With no arguments or IDs,
  prints the full history oldest-first.

  Filter Options ~

    `--after {date}`          entries after {date} (natural language)
    `--before {date}`         entries before {date}
    `--contains {str}`        entries whose command contains {str}
    `--starts-with {str}`     entries whose command starts with {str}
    `--ends-with {str}`       entries whose command ends with {str}
    `--matches {regex}`       entries matching extended regex {regex}
    `--duration-gt {dur}`     entries with runtime longer than {dur}
    `--duration-lt {dur}`     entries with runtime shorter than {dur}
    `--lines-gt {n}`          entries with more than {n} lines
    `--lines-lt {n}`          entries with fewer than {n} lines
    `--with-status {code}`    entries whose command exited with {code}
    `--with-token {uuid}`     entries with the given unique token
    `--in-dir {path}`         entries run from {path} (exact match; "." expands)
    `--limit {n}`             return at most {n} results

  Any filter option can be negated by placing `--not` immediately before it:

      `hist --not --with-status 0`     commands that exited non-zero
      `hist --not --contains git`      commands not containing "git"
      `hist --not --in-dir .`          commands run outside current directory

  Output Options ~

    `-n`           suppress history IDs
    `-r`           newest-first order (default is oldest-first)
    `--count`      print only the count of matching entries
    `--json`       output as JSON (includes command, timestamp, runtime,
                   cwd, status, and token fields)

  Mutation Options ~

    `--delete`          delete matching entries
    `--restore`         restore previously deleted entries
    `--import {source}` import history from another shell (see below)
    `--ex`              operate on ex-mode history instead of shell command history

  Providing numeric arguments restricts results to those specific IDs:

      `hist 42 43`    print entries with IDs 42 and 43

  Dates accept natural language: "yesterday", "2 hours ago", "last
  monday", ISO 8601, etc. Durations: "5s", "2 minutes", "1 hour".

  Importing History ~

  `--import {source}` imports history from another shell's history file.
  {source} can be a shell name or an explicit file path:

    `bash`    ~/.bash_history   (plain text or with HISTTIMEFORMAT timestamps)
    `zsh`     ~/.zsh_history    (plain text or extended format)
    `fish`    ~/.local/share/fish/fish_history
    any path  treated as a file to auto-detect and import

  The format is detected automatically from the file contents. After import,
  entries are sorted by timestamp so imported history interleaves correctly
  with existing entries.

  Examples:

      `hist`                                    full history
      `hist --contains git`                     entries containing "git"
      `hist --after "yesterday"`                entries since yesterday
      `hist --limit 20 -n`                      last 20 commands, no IDs
      `hist --delete --contains foo`            delete entries containing "foo"
      `hist --not --with-status 0`              failed commands
      `hist --in-dir . --after "1 hour ago"`    recent commands in current dir
      `hist --import bash`                      import from ~/.bash_history
      `hist --import zsh`                       import from ~/.zsh_history
      `hist --import ~/old_history`             import from explicit path

KILL                                             *builtin-kill* *kill*

  Send a signal to one or more targets. Default signal is SIGTERM.

LOCAL                                            *builtin-local* *local*

  `local [name[=value] ...]`

  Declare variables scoped to the current block. Local variables
  shadow any outer variable of the same name and are discarded when
  the enclosing scope exits. Unlike bash, where `local` only applies
  in functions, shed binds `local` to whatever scope frame is current.

  Scopes that hold `local` variables ~

    Function bodies                     variables die on function return
    Brace groups `{ ... }`              variables die on block exit
    Compound commands (`if`, `for`,...) variables die on exiting the body

  With `name=value`, declares and sets in one step. With just {name},
  declares with an empty value. Multiple declarations per call are
  allowed.

  With no arguments, lists all variables in the current scope.

  Combine with |builtin-defer| for save-and-restore patterns: `local`
  captures the value, `defer` registers the restore, and both vanish
  together when the scope ends.

  Examples:

      `local x=10`
      `local a b c`
      `local`           list scope variables

MAP                                              *builtin-map* *map* *maps*

Maps are shell-native tree structures for associating keys with values.
They support nested paths, dynamic evaluation, arrays, and JSON
serialization. Maps are particularly useful for building completion
trees, configuration hierarchies, and structured data.

1. Setting Values ~

  `map {name}.{path}={value}`

  Assign {value} to the node at {path} within map {name}. Intermediate
  branches are created automatically if they do not exist.

  Examples:

      `map colors.red=#ff0000`
      `map config.editor.theme=dark`
      `map git.remote.origin=https://github.com/user/repo`

  If the map {name} does not yet exist, it is created.

  Dynamic Leaves                                 *map-dynamic*

  `-F`

      Mark the value as a dynamic leaf. Instead of storing a static
      string, the value is treated as a command and evaluated via
      command substitution each time the leaf is read.

      `map -F git.branches='git branch --format="%(refname:short)"'`

      Each time `map git.branches` is accessed, the command runs and
      returns fresh output. This is ideal for completion candidates
      that change over time (branches, tags, containers, etc.).

  JSON Import                                    *map-json*

  `-j`

      Parse the value as JSON. Objects become branches, arrays become
      array nodes, and strings become static leaves.

      `map -j mymap='{"key": "val", "nested": {"a": 1}}'`

  Local Scope                                    *map-local*

  `-l`

      Create the map in the current scope only. It is removed when the
      scope exits (e.g. when a function returns).

      `map -l scratch.tmp=working`

2. Reading Values ~

  `map {name}`
  `map {name}.{path}`

  Print the value at the given path. If the node is a branch, the
  output is JSON. If it is a static leaf, the raw string is printed.
  If it is a dynamic leaf, the stored command is evaluated and its
  output is printed.

  Options:

    `-j`         Force JSON output even for leaf nodes.
    `--pretty`   Pretty-print JSON output with indentation.
    `-k`         Print only the keys of the node (children of a branch,
                 or elements of an array).

  Examples:

      `map config.editor.theme`       # prints: dark
      `map -j config`                 # prints full JSON tree
      `map --pretty config`           # prints indented JSON
      `map -k config.editor`          # prints keys under editor

3. Removing Values ~

  `map -r {name}`
  `map -r {name}.{path}`

  Remove the node at the given path. If only {name} is given, the
  entire map is removed. If a path is given, only that subtree is
  removed; parent branches are preserved.

4. Node Types ~
                                                 *map-nodes*

  `Branch`         A named collection of child nodes. Accessed by
                   string keys via dot-path notation.

  `Array`          An ordered list of child nodes. Accessed by numeric
                   index via dot-path notation.

  `StaticLeaf`     A string value, returned as-is when read.

  `DynamicLeaf`    A command string, evaluated via command substitution
                   when read. Created with `-F`.

  Leaf nodes are automatically promoted to branches when a sub-path is
  set beneath them.

5. Paths ~
                                                 *map-paths*

  Paths use `.` as a separator. Each segment is a key into the current
  node. For branches, the segment is a string key. For arrays, it is
  a numeric index.

      `map data.users.0.name=alice`
      `map data.users.1.name=bob`
      `map data.users.0.name`          # prints: alice

  Path segments are expanded before lookup, so variables work:

      `map mymap.$key=value`

6. Completion Trees ~
                                                 *map-completion*

  Maps are well-suited for defining completion hierarchies. Combine
  static branches for subcommands with dynamic leaves for
  context-sensitive candidates:

      `map -F git.checkout='git branch --format="%(refname:short)"'`
      `map -F git.switch='git branch --format="%(refname:short)"'`
      `map git.remote.add=`
      `map -F git.remote.remove='git remote'`
      `map git.commit=`
      `map git.push=`

      `map -k git`                 # checkout switch merge remote ...
      `map git.checkout`           # evaluates: git branch ...

  Wildcard Keys                                  *map-wildcard*

  Use `%` as a path segment to create a wildcard key. A wildcard
  matches any key that does not have an exact static match, allowing
  trees to describe commands with freeform arguments (filenames,
  branch names, etc.) that have further structure beneath them.

      `map git.checkout.%.--force=`
      `map git.checkout.%.--track=`

      `map -k git.checkout.my-branch`   # --force --track

  When `-k` is used on a leaf node (no children), the value is
  printed instead of keys and the exit status is set to 1. This
  allows traversal loops to detect the end of a path:

      while keys=$(map -k $path); do
          # pick next key, continue traversal
          path="$path.$key"
      done
      # loop exited: $keys holds the leaf value

7. JSON Interop ~
                                                 *map-json-interop*

  Maps can be imported from and exported to JSON:

      `map -j tree='{"bin":{"ls":{},"cat":{}},"etc":{}}'`
      `map --pretty tree`

  This allows completion trees and configuration maps to be stored in
  files or generated by external tools.

8. Options Summary ~
                                                 *map-options*

  `-r`         Remove the specified node or entire map
  `-k`         Print only the keys of a node
  `-j`         Parse value as JSON (set) or force JSON output (read)
  `--pretty`   Pretty-print JSON output
  `-F`         Store value as a dynamic leaf (command evaluated on read)
  `-l`         Create map in local scope only

POPD                                             *builtin-popd* *popd*

  Remove an entry from the directory stack. By default pops the top

    `-C`  noclobber   disallow `>` from overwriting existing files

STASH                                            *builtin-stash* *stash*

  Save and recall command snippets. Stash operates in two modes:

  `stash --list [--named | --stack]`             *stash-list-filter*

      List stashes, optionally filtering to only named or only

  Unary string operators ~

    `-n {s}`    string is non-empty
    `-z {s}`    string is empty

  Binary operators ~

    `{a} == {b}`    strings equal
    `{a} != {b}`    strings not equal
    `{a} =~ {b}`    {a} matches extended regex {b}
    `{a} -eq {b}`   integers equal
    `{a} -ne {b}`   integers not equal
    `{a} -gt {b}`   integer greater than
    `{a} -lt {b}`   integer less than
    `{a} -ge {b}`   integer greater than or equal
    `{a} -le {b}`   integer less than or equal

  Logical operators ~

    `{e1} && {e2}`   conjunction
    `{e1} || {e2}`   disjunction
    `! {e}`          negation

  POSIX character classes are supported in `=~` patterns:
  `[[:alpha:]]`, `[[:digit:]]`, `[[:space:]]`, etc.

  Examples:

      `[[ -f /etc/passwd ]]`
      `[[ $x -gt 10 && $x -lt 20 ]]`
      `[[ $name =~ ^[A-Z] ]]`
      `[[ -z $reply ]] && echo "empty"`

TRAP                                             *builtin-trap* *trap*

  `trap [{action}] {target} ...`

  Register {action} to run when the shell receives {target}. {action}
  is a shell command string that runs in the current environment.
  Multiple targets may be given.

  Use `-` as {action} to reset a target to its default disposition.
  Use `""` to explicitly ignore a signal.

  Targets ~

    `EXIT`    fires when the shell exits normally
    `ERR`     fires when any command returns non-zero
    Signal names (without `SIG` prefix): `INT`, `TERM`, `HUP`,
    `QUIT`, `USR1`, `USR2`, `CHLD`, `PIPE`, `ALRM`, `WINCH`,
    `TSTP`, `CONT`, and more.

  Examples:

      `trap 'echo bye' EXIT`
      `trap 'cleanup' INT TERM`
      `trap - INT`              restore default SIGINT
      `trap '' HUP`             ignore SIGHUP

TIMES                                            *builtin-times* *times*

  `times`

  Print accumulated user and system CPU times for the shell and for all
  waited-on child processes. Output format:

      {shell_user}m{s}s  {shell_sys}m{s}s
      {child_user}m{s}s  {child_sys}m{s}s

  First line: shell process time. Second line: total child process time.

TYPE                                             *builtin-type* *type*

  `type {name} ...`

  Describe how the shell would interpret each {name} if used as a
  command. Reports the first match in lookup order:

    1. function - prints name, source file, and line/column
    2. alias    - prints the alias definition and source location
    3. builtin  - prints "{name} is a shell builtin"
    4. keyword  - prints "{name} is a shell keyword"
    5. command  - prints the full path found in `$PATH`

  Returns exit status 1 if a name is not found.

  Examples:

      `type echo`       echo is a shell builtin
      `type ls`         ls is /usr/bin/ls
      `type if`         if is a shell keyword
      `type ll`         ll is an alias for 'ls -la' ...

ULIMIT                                           *builtin-ulimit* *ulimit*

  `ulimit [-n N] [-u N] [-s N] [-c N] [-v N]`

  Set soft resource limits for the current shell and processes it
  spawns. Each flag takes a numeric value. The hard limit is left
  unchanged; it is an error to set a soft limit above the hard limit.

  `-n {N}`   maximum number of open file descriptors
  `-u {N}`   maximum number of user processes
  `-s {N}`   maximum stack size in bytes
  `-c {N}`   maximum core dump file size in bytes (0 = disabled)
  `-v {N}`   maximum virtual memory (address space) in bytes

  Multiple flags may be combined in one call.

  Examples:

      `ulimit -n 4096`      raise file descriptor limit
      `ulimit -c 0`         disable core dumps

UMASK                                            *builtin-umask* *umask*

  `umask [-S] [{mask}]`

  Get or set the file creation mask. The mask is subtracted from the
  permissions that programs request when creating new files.

  With no arguments, prints the current mask as a 4-digit octal number.

  `-S`   print the mask in symbolic form (`u=rwx,g=rx,o=rx`)

  {mask} forms ~

    Octal (e.g. `022`)   set the mask directly
    Symbolic             modify the mask using `chmod`-style notation:
                         `u`, `g`, `o`, `a` as who;
                         `r`, `w`, `x` as permissions;
                         `=`, `+`, `-` as operators.
                         Multiple clauses separated by `,`.

  Examples:

      `umask`              print current mask (e.g. 0022)
      `umask -S`           print symbolically (e.g. u=rwx,g=rx,o=rx)
      `umask 077`          no permissions for group or other
      `umask u=rwx,g=rx,o=rx`   set symbolically
      `umask a+r`          allow read for all (clear read mask bit)
      `umask g-w`          deny write for group (set write mask bit)
