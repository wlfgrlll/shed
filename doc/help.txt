#SHED MANUAL#

Shed is an experimental UNIX command interpreter (shell) usable as both an
interactive login shell and as a shell script command preprocessor.
Shed combines the functionality of `bash` with the UX and customizability of `vim`.
Features include:
* A hackable line editor similar to `zsh`'s `zle`
* Programmable, dynamic prompts
* Custom tab completion
* An event hook system
* An IPC socket for async workflows
* Fuzzy history search/tab completion
* Extensive, navigable documentation via `help`

Press `Tab` to follow cross-reference links in the help pages. Visible links
will be given a hint with a letter, typing that letter follows the link.

Cross-reference links can also be `clicked` directly with the mouse.

==============================================================================
1. Available Topics                            *help-topics*

  `help autocmd`      Autocmd hooks for shell events               |autocmd|
  `help builtin`      Builtin command reference                    |builtin|
  `help ex`           Ex mode (colon) commands                     |ex|
  `help jobs`         Job control and background processes         |jobs|
  `help keybinds`     Vi mode keys, motions, text objects          |keybinds|
  `help prompt`       Prompt configuration, PS1, PSR, echo -p      |prompt|
  `help scripting`    Shell scripting language reference           |scripting|
  `help variables`    Variables, assignment, arrays, attributes    |variables|
  `help arith`        Arithmetic expansion: `$(( ))` syntax          |arith|
  `help commands`     Common command cheat sheet                   |commands|
  `help glob`         Pathname expansion and wildcards             |glob|
  `help param`        Parameter expansion: `${}` operators           |param|
  `help redirect`     Redirection, pipes, heredocs                 |redirect|

==============================================================================
2. Navigating Help                             *help-nav*

  `help {topic}`      Open a help page by name. Prefix matches work,
                      so `help key` opens |keybinds|.

  `help {tag}`        Jump directly to a tagged section. Tags are the
                      words in the top right of each section.

                      Examples:
                        `help help-nav`         jumps to this section
                        `help prompt-escapes`   jumps to |prompt-escapes|
                        `help heredoc`          jumps to |heredoc|

  Help pages are displayed in an interactive pager.      *help-pager*

  2.1 Scrolling                                  *help-scroll*

      `j` / {Down}        Scroll down one line
      `k` / {Up}          Scroll up one line
      `d`               Scroll down 15 lines
      `u`               Scroll up 15 lines
      `g`               Jump to top of page
      `G`               Jump to bottom of page
      `q` / {Esc}         Quit the pager

      Mouse scrolling is also supported.

  2.2 Page Navigation                            *help-nav*

      `h` / {Left}        Navigate to previous open page
      `l` / {Right}       Navigate to next open page

      Mouse back/forward buttons also work for page navigation.

  2.3 Search                                     *help-search*

      `/`{pattern}        Search forward
      `?`{pattern}        Search backward
      `n`               Jump to next match
      `N`               Jump to previous match

      Search results are highlighted as you type.

  2.4 Cross-Reference Navigation                 *help-hints*

      {Tab}               Enter hint mode

      In hint mode, each visible cross-reference is labelled with
      a single character. Press that character to follow the link
      and open the referenced help page. Press {Esc} to cancel.

      Cross-references can also be clicked directly with the mouse.
